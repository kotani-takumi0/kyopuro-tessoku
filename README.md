# 競技プログラミングの鉄則 38問（C++）

## 目的
- アルゴリズム・データ構造の基礎力を身につける  
- C++ の実装力（スピード・正確性）を強化する  
- コーデディング面接に必要な基礎力を習得する  
- 実務や ML/RAG 開発に使える実装力を獲得する  

## 対象
『競技プログラミングの鉄則』のうち、★1〜★3 相当の 38 問を扱う。

## 学習方針
- スピードより再現性を優先する（翌日もう一度書ける状態を目標）
- 解けなければすぐに解説を読む（時間を浪費しない）
- C++テンプレートを使い、書き始めの負荷を下げる
- 各問題の振り返りを memo.md に必ず書く

## コミットルール
1問につき最低3コミットとする。

### 1. First attempt（初回実装）
- まだACしていなくてもよい  
- 方針に沿って一度最後まで書いた段階でコミット  
- 例：  
  `git commit -m "feat: solve q05 first attempt"`

### 2. Fix / Refactor（修正）
- 解説を読んで理解し、バグ修正やロジック整理をした段階でコミット  
- 例：  
  `git commit -m "fix: correct boundary condition in q05"`

### 3. Memo / Docs（振り返り）
- memo.md に学んだ点・失敗箇所・改善点を追記してコミット  
- 例：  
  `git commit -m "docs: add notes for q05"`

## 学習ルール
- 毎日最低1コミット  
- 1問につき「初回実装→修正→振り返り」の流れを必ず行う  
- 類似問題を見たときに書き直せる状態を目指す

## 進捗管理
- 完了した問題数を README に随時記録する  
- 学習ログは log/YYYY-MM-DD.md として残す
